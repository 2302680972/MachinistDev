# 图片和动图的实现指引

## 本篇范围

- UI 图片与帧动画的实现方式
- 动图拆帧与播放策略

## 不包含

- UI 引用管理与运行时规则（见"canvas-component-guide"）
- 动画插值与缓动（见"canvas-animation-mechanism"）

## 显示图片

使用 `UIRawImage` 组件。设置 `picGuid` 属性为图片资源的 GUID，设置 `color` 属性控制颜色叠加和透明度。

## 九宫格

`UIRect` 和 `UIButton` 支持九宫格。需要设置：
- 九宫格图片的 GUID
- 左下边距（Vector3：x=左边距，y=下边距）
- 右上边距（Vector3：x=右边距，y=上边距）

九宫格实现图片拉伸时边缘不变形。

## 动图实现

平台不直接支持 GIF 格式。需要通过脚本逐帧切换图片实现。

实现思路：
1. 将动图拆分为单帧图片，按命名规则（如 frame_0, frame_1...）
2. 获取所有帧的 GUID，存入 List
3. 维护当前帧索引、帧间隔计时器
4. 在 `update` 中累加计时器，达到间隔时切换到下一帧
5. 循环播放时索引回到 0，单向播放时到达末帧结束

## 帧序列

帧序列图片需要连续命名，脚本按顺序加载 GUID。帧数量、帧间隔（fps）可配置。

## 注意事项

- 图片 GUID 必须正确，否则显示为空白或默认图
- 帧序列必须连续，不能有缺失
- 控制帧率避免过高的更新频率影响性能
- 大量同时播放的动图考虑使用对象池
- 静态图片优先使用单图，不要拆成帧序列
