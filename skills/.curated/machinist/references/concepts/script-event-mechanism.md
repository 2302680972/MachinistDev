# 脚本与事件机制

## 本篇范围

- 事件脚本与自定义脚本的区分
- 私有权限与动态调用
- 初始化脚本推荐写法

## 不包含

- TS 视图语法与写回规则（见"TS 视图编写规范"）
- 机械生成器注入机制（见"机械复用机制指引"）

## 事件 Event

事件是平台内部变化的触发点：

- 地图生命周期（进入地图即游戏启动）
- 机械对象生命周期（创建/销毁）
- 交互事件：零件碰撞/收到弹幕/每帧更新
- 大地图通知

### 事件特性

- 事件类型已预定，参数固定，无返回值
- 触发时机由底层决定
- 挂载的脚本按列表顺序轮流触发
- 任何调用栈追溯其根源，必然起源于某个事件

## 脚本 Script

所有游戏逻辑均由脚本完成。脚本必须依附于零件，不能直接挂在 Map 或 Mech 上。

### 事件脚本

- 由底层引擎根据事件自动调用
- 不可手动调用
- 职责是响应事件并调度其他自定义脚本

### 自定义脚本

- 可被事件脚本调用，也可被其他自定义脚本调用
- 支持"私有"权限选项：
  - 标记为私有时，不出现在其他零件的可选方法列表中
  - 同一零件内部的其他方法可正常静态调用私有方法
  - 动态调用无法访问私有方法
- 未标记私有的脚本可被其他零件通过静态或动态调用访问

### 动态调用

类似"反射"能力：

- 通过"函数签名 + 目标零件引用"调用目标零件上的自定义方法
- 持有 Mech 变量时，可仅凭签名直接调用该机械核心的方法
- 私有方法不可被动态调用访问

## 递归禁止

脚本语言完全禁止递归：

- 无论直接递归、间接递归、还是通过动态调用形成的递归调用链
- 检测到递归调用时会被引擎跳过
- 需要重复行为时用循环或显式展开

## 初始化脚本推荐写法

### 地图初始化

- 游戏整体初始化脚本推荐挂在地图相关零件上
- 通过该零件的启动事件完成整张地图的初始化
- 推荐设置唯一初始化入口，内部按可预测顺序串联各部分

### 机械初始化

- 需要放置时做额外初始化的机械，可在核心零件上设置初始化脚本
- 每次被放置到地图上时执行机械层面的初始化逻辑

### 通用逻辑复用

- 一组机械的通用逻辑放到机械产生器
- 特定机械文件特有逻辑放在机械文件的核心
